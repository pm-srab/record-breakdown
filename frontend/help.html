คู่มือการใช้งานระบบ (System Guide)

ระบบติดตามและซ่อมบำรุงเครื่องจักร (Maintenance Monitoring System) แบ่งหน้าการทำงานหลักออกเป็น 5 ส่วน ดังนี้:

1. Record (แจ้งซ่อม)

ใช้สำหรับบันทึกแจ้งยอดขัดข้องของเครื่องจักรเข้าสู่ระบบ โดยผู้บันทึกต้องระบุไลน์การผลิต, รหัสเครื่องจักร, อาการขัดข้อง เพื่อให้ทีมช่างซ่อมบำรุงเข้าตรวจสอบ

2. Dashboard (แดชบอร์ด)

แดชบอร์ดสรุปวิเคราะห์สถิติสถานะงานซ่อมแบบ Real-time รวบรวมข้อมูลระยะเวลา Breakdown และชั่วโมงสะสมของการหยุดไลน์การผลิตของเครื่องจักรแต่ละเครื่อง

3. ปิดงาน / 7 Mean

หลังการซ่อมบำรุงเสร็จสิ้น ทำการปิดงานในส่วนนี้ โดยกรอกข้อมูลวันที่ซ่อมเสร็จ, วิธีแก้ไข, และ 7 Mean เพื่อเก็บประวัติ

4. Spare Part (คลังอะไหล่)

ส่วนงานจัดการและตรวจสอบสต็อกชิ้นส่วนอะไหล่ สามารถทำรายการเบิกจ่ายชิ้นส่วนอะไหล่ไปใช้งาน เพื่อให้ระบบคำนวณและตัดยอดสต็อกคงเหลือในชีททันที

5. History (ประวัติแจ้งซ่อม)

แสดงประวัติการแจ้งซ่อมย้อนหลังทั้งหมดของระบบอย่างละเอียด สามารถค้นหาและคัดกรองข้อมูลเพื่อวิเคราะห์ประสิทธิภาพในการซ่อมบำรุงของไลน์การผลิต

หากมีปัญหาต้องเข้าไปแก้ตรงไหน (Troubleshooting)

เมื่อพบเจอปัญหาขัดข้องหรือต้องการปรับแก้โปรแกรม คุณสามารถดำเนินการตามขั้นตอนของคู่มือโดยละเอียดดังนี้:

หากต้องการปรับแต่งหรือแก้ไขโค้ดการทำงานโปรแกรมซ่อมบำรุง (How to edit the code)
หากมีความรู้ทางเทคนิคและต้องการเข้าไปแก้ไขโค้ดโปรแกรมของแต่ละหน้าเว็บ สามารถทำได้ด้วยตนเองผ่าน GitHub ตามขั้นตอนต่อไปนี้:
  1. ล็อกอินและเข้าไปยังคลังเก็บซอร์สโค้ดโปรแกรม (GitHub Repository) ที่นี่: https://github.com/pm-srab/record-breakdown
  2. เมื่อเข้าลิงก์แล้ว จะพบหน้าต่างรวมไฟล์โปรแกรมระบบซ่อมบำรุงดังรูปภาพประกอบด้านล่างนี้ โดยที่แต่ละหน้าเว็บหลักจะควบคุมแยกผ่านไฟล์ HTML อย่างชัดเจน:
  1. คลิกเลือกไฟล์ HTML ที่สอดคล้องกับหน้าเว็บที่ต้องการจะแก้ไข (เช่น เลือก index.html หากต้องการแก้หน้าหลัก)
  2. คลิกที่ไอคอน ดินสอ (Edit this file) บริเวณมุมบนขวา เพื่อเข้าสู่หน้าจอแก้ไขโค้ด (Source Code Editor) ดังรูปภาพด้านล่าง:
  1. เมื่อทำการแก้ไขซอร์สโค้ดเรียบร้อยแล้ว ให้คลิกปุ่มสีเขียว "Commit changes..." ที่บริเวณมุมบนขวาสุดของตัวเว็บ (ดังรูปภาพประกอบด้านล่าง):
  1. เมื่อหน้าต่างยืนยันปรากฏขึ้น ให้กรอกรายละเอียดสั้นๆ แล้วกดปุ่ม **Commit changes** เพื่อบันทึกข้อมูลและให้ระบบอัปเดตออนไลน์แบบ Real-time
⚠️ ข้อสำคัญ: หากปัญหาหรือการแก้ไขมีความซับซ้อนเกินกว่าที่จะแก้ไขเองได้ กรุณาหยุดแก้ไขและติดต่อทีมพัฒนาโปรแกรม (K&A Software House) ด้านล่างทันที
โปรแกรมเชื่อมต่อ Google Sheet ไหนบ้าง และถ้าเกิดปัญหาข้อมูลไม่ตรงกัน / ไม่ซิงค์ต้องทำอย่างไร?
โปรแกรมระบบติดตามงานซ่อมนี้ใช้ Google Sheets ในการจัดเก็บฐานข้อมูลหลักแยกออกจากกันเป็น 2 ไฟล์หลัก เพื่อความปลอดภัยและเป็นระบบ:
📂 รายการเชื่อมฐานข้อมูล Google Sheets (Database Links):
  • ฐานข้อมูล Dashboard & History (ผลการแจ้งซ่อม): ใช้เก็บผลงานซ่อม วิเคราะห์ 7 Mean และประวัติแจ้งซ่อมทั้งหมด
    🔗 ลิงก์ชีท: Google Sheets - ชีทบันทึกและผลแจ้งซ่อมหลัก
  • ฐานข้อมูล Spare Part (สต็อกคลังอะไหล่): ใช้เก็บบริหารจัดการยอดอะไหล่และสถิติตัดยอดเบิกจ่ายอะไหล่ในระบบ
    🔗 ลิงก์ชีท: Google Sheets - ชีทบริหารคลังสต็อกและรายการอะไหล่
🔄 ขั้นตอนแนะนำในการซิงค์ข้อมูลชุดล่าสุด (Sync & Clear Browser Cache):

หากข้อมูลที่เพิ่งแก้ไขบน Google Sheets หรือผลลัพธ์ในหน้าเบราว์เซอร์ไม่ตรงกัน (Out of Sync):

  • สามารถกดคีย์ลัด Ctrl + F5 บนหน้าเบราว์เซอร์พร้อมกัน เพื่อรีเซ็ตหน้าเว็บแบบ Hard Reload
  • หรือคลิกที่ปุ่มสีแดง "ล้างแคชและซิ้งใหม่" บริเวณเมนูด้านซ้ายล่างสุดของตารางแสดงผลในหน้า Dashboard (ดังรูปภาพประกอบด้านล่าง) เพื่อล้างแคชเก่าในหน่วยความจำของหน้าเบราว์เซอร์ และดึงชุดข้อมูลล่าสุดจาก Google Sheet ทันที:
